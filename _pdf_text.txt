Session key (JWT token) is required for accessing Public REST API. It must be present in authorization header. The key is  valid for one hour. Once it expires, the new key must be generated. There is a limit of how many session keys can be  active at a time. If this limit is reached, the oldest session key is revoked and cannot be used anymore. Public REST API for dealing with cards provides an ability to have a cards dictionary with the following formats:  Overview 2.4.1.  UID   - allows to specify UID as card value in raw format. It allows to assign this card in different hardware (wiegand)  formats using a specific algorithm. The algorithm is - take the needed number of bytes from the beginning (if there  are not enough bytes provided, then fill with zeros before provided bytes) and reverse it.  Example:  value - AABBCCDDEE  format -   W26   => value to assign - CCBBAA  format -   W34   => value to assign - DDCCBBAA  value - AABB  format -   W26   => value to assign - BBAA00  format -   W34   => value to assign - BBAA0000  MEMORY   - allows to provide a memory block that can be used for assigning this card in different formats.  The algorithm is - take the needed number of bytes from the beginning  Example:  value - AABBCCDDEE  format -   W26   => value to assign - AABBCC  format -   W34   => value to assign - AABBCCDD  Specific wiegand format ( W26, W34, W36, W37, W42, W58DEC, W58 ) - just perceives the value as ready to assign.
